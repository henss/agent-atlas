# @agent-atlas/markdown

Markdown rendering package for generated agent-facing docs.

Implemented outputs:

- root atlas view
- domain cards
- workflow cards
- component cards
- resource/document cards when visible in the selected profile
- verification guide
- optional root README entrypoint

The README renderer infers a human-first default orientation from the graph:
repository purpose, major workflows, common CLI entry points, command groups,
common docs, where durable state lives, and verification scopes. CLI entry
points come from generated command metadata when present and name/summary
inference otherwise. Agent-specific startup rules should stay in `AGENTS.md`;
`repository.metadata.readme` can still override or extend sections when a repo
needs project-specific wording.

Generated files start with:

```md
<!-- Generated by Agent Atlas. Do not edit directly. -->
```

The renderer supports `public`, `private`, and `company` profiles. Public output excludes entities marked `private`, `internal`, or `restricted`, and redacts private URI schemes if a public-visible entity accidentally includes one.
